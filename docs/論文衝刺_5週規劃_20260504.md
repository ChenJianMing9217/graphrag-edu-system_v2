> 本文件追蹤從上線完成到投稿草稿的 5 週執行路徑。
> 所有 deliverable 對應論文章節，非單純工程任務。

---

## 規劃原則

1. **Critical path：先穩定上線、再做研究**。治療師測試會產生新資料，這是論文最珍貴的素材，不能因實驗 push 不穩。
2. **離線分析優先**：所有不影響系統運作的事（ablation, 重訓, 評估）先做。
3. **非破壞性 instrument**：log 補充寫在現有結構外，不改主流程。
4. **每週一個明確 deliverable**：每階段都產出可放進論文的東西。

## Phase 3：段 3 Tiny Attention（D+17 ~ D+23）

### Day 17-18：實作 ContinuityTracker
```python
class ContinuityTracker(nn.Module):
    # 1 layer, 2 heads, dim=64
    # ≈ 50K params
    def forward(self, turn_seq):
        # turn_seq: B × T × (q_emb + features)
        h = self.input_proj(turn_seq)
        h = self.transformer(h, mask=causal)
        return self.head(h[:, -1, :])
```

## Reviewer 必問題 + 答案準備

| Q | A |
|---|---|
| 為什麼不用 LLM？ | latency 380x + 不可解釋 + 我們是 LLM 的補充非對立 |
| 591 sample 太少？ | 醫療對話資料受倫理限制，採半合成 + 治療師 audit |
| 8 維特徵 ad-hoc？ | Tier 1-4 對應明確 sub-problem，每維用 ablation 證明 |
| domain-based 任意？ | retrieval 是 domain-indexed + user mental model + sub-problem decomposition |
| 與 TripPy 等 DST 差別？ | 不追蹤 slot value，追蹤 retrieval scope |
| 為什麼挑早療？ | knowledge-intensive 又 emotion-sensitive 領域 |
